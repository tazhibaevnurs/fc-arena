# Пошаговый план: запуск монетизации (подписки + реклама AdSense и РСЯ)

**Цель:** чтобы деньги поступали от подписок PRO и от просмотров рекламы (Google AdSense и Яндекс.Реклама / РСЯ).

---

## Часть A. Реклама (AdSense + РСЯ)

### Шаг A1. Подготовка сайта к модерации

1. **Домен и SSL**  
   Сайт должен быть доступен по постоянному домену с HTTPS (обязательно для AdSense и РСЯ).

2. **Контент и политики**  
   - Есть страницы с текстом (главная, турниры, подписка, профиль).  
   - Добавьте страницу «О проекте» / «Правила» и ссылку в футере (требования рекламных сетей).  
   - Убедитесь, что в шаблонах нет запрещённого контента (в т.ч. в пользовательских никах/названиях — при необходимости цензурируйте).

3. **Политика конфиденциальности и cookies**  
   Для AdSense и РСЯ нужна страница с политикой конфиденциальности и, при необходимости, баннер/ссылка про использование cookies (особенно для ЕС).

---

### Шаг A2. Google AdSense

1. Регистрация: [google.com/adsense](https://www.google.com/adsense).  
2. Добавление сайта и верификация (мета-тег или DNS).  
3. После одобрения:
   - Создайте блоки: например, «Верх (728×90 или адаптив)», «Боковая панель», «В контенте».
   - Скопируйте **Publisher ID** (формат `ca-pub-XXXXXXXXXXXXXXXX`) и **ID слотов** (формат `XXXXXXXXXX`).
4. В админке Django: **Турниры → Ad settings** (или единственная запись AdSettings):
   - Включите **Google AdSense** и **Глобально реклама**.
   - **AdSense client id:** `ca-pub-XXXXXXXXXXXXXXXX`.
   - **Слоты:** заполните поля top, sidebar, bottom, in_content соответствующими ID слотов.
5. Сохраните. Реклама уже подключена через `ad_tags` в `base.html` и на страницах турнира — блоки начнут показываться после заполнения настроек.

---

### Шаг A3. РСЯ (Яндекс.Реклама)

1. Регистрация: [yandex.ru/adv](https://yandex.ru/adv/) → Рекламная сеть (РСЯ).  
2. Добавление сайта и модерация.  
3. Создайте блоки РСЯ (тип «Код блока»), получите **ID блоков** (формат `R-A-XXXXX-X`).  
4. В админке Django: **Ad settings**:
   - Включите **РСЯ (Яндекс)**.
   - **РСЯ: ID блока** — укажите один или несколько ID (в коде используется один блок на сайт; при необходимости можно доработать шаблон под несколько блоков).
5. Скрипт РСЯ: обычно подключается один раз на сайт (проверьте документацию Яндекса). Если нужен отдельный скрипт инициализации блоков по ID — добавьте его в `base.html` (или в `ad_tags`), чтобы блоки заполнялись.

---

### Шаг A4. Проверка показа и доходов

- Зайдите на сайт **без** PRO-подписки: должны отображаться и AdSense, и РСЯ в предусмотренных местах (header, sidebar, in-content).  
- С аккаунтом PRO реклама не показывается (`show_ads` / `ads_manager` скрывают блоки).  
- В кабинетах AdSense и Яндекс.Реклама через 1–2 дня появятся данные по показам и доходам.

---

## Часть B. Подписки (деньги за PRO)

### Шаг B1. Подключение платёжных систем

1. **2Checkout (Verifone)**  
   - Активируйте аккаунт, получите **Seller ID** и **Secret Word**.  
   - В .env: `TWOCHECKOUT_SID=...`, `TWOCHECKOUT_SECRET_WORD=...`.  
   - Для тестов: `TWOCHECKOUT_DEMO=1`.  
   - В настройках 2Checkout укажите:  
     - Return URL: `https://ваш-домен/accounts/subscription/success/`  
     - IPN URL: `https://ваш-домен/accounts/payment/2checkout/ipn/`  
   - После оплаты подписка активируется по IPN и по return URL.

2. **Robokassa**  
   - Активируйте магазин, получите **MerchantLogin**, **Password1**, **Password2**.  
   - В .env: `ROBOKASSA_LOGIN=...`, `ROBOKASSA_PASSWORD1=...`, `ROBOKASSA_PASSWORD2=...`.  
   - Result URL: `https://ваш-домен/accounts/payment/robokassa/result/`  
   - Success/Fail URL — как в коде (редирект на профиль и на страницу подписки).  
   - Для тестов: `ROBOKASSA_TEST=1`.

---

### Шаг B2. Юридическая и налоговая готовность

- Определите страну приёма платежей и применимое право.  
- Подготовьте оферту/договор подписки и возвраты (отмена в любой момент уже указана на странице).  
- Настройте вывод средств и отчётность (2Checkout/Robokassa обычно дают отчёты и выплаты на банк/карту).

---

### Шаг B3. Контроль конверсии

- В .env уже можно задать `GA4_MEASUREMENT_ID` и `YANDEX_METRIKA_ID`.  
- События воронки: `subscription_page_view`, `checkout_click`, `payment_success`, `payment_fail` — используйте их в целях GA4/Метрики для расчёта конверсии и донастройки страницы подписки.

---

## Часть C. Язык при просмотре из-за рубежа

**Да, язык может меняться при просмотре из другой страны.**

В проекте включён **LocaleFromIPMiddleware**:

- При **первом** заходе, если у пользователя **нет** cookie `django_language`, определяется страна по IP (сервис ip-api.com).
- По стране выставляется язык из списка:
  - США, Великобритания, Австралия, Канада и др. → **English (en)**
  - Германия, Австрия, Швейцария → **Deutsch (de)**
  - Испания, Мексика, Аргентина и др. → **Español (es)**
  - Франция, Бельгия и др. → **Français (fr)**
  - Португалия, Бразилия → **Português (pt)**
  - Украина → **Українська (uk)**
  - Казахстан → **Қазақша (kk)**
  - Россия, Беларусь, Кыргызстан и др. → **Русский (ru)**
- Выбранный язык сохраняется в cookie на год; пользователь может вручную сменить язык в переключателе в шапке сайта.

**Важно:**  
Сейчас в шаблонах большая часть текста захардкожена по-русски. Чтобы интерфейс реально отображался на выбранном языке:

1. Оберните строки в шаблонах в `{% trans "Текст" %}` или `{% blocktrans %}...{% endblocktrans %}`.  
2. Выполните `python manage.py makemessages -l en` (и для de, es, fr и т.д.).  
3. Заполните переводы в файлах `locale/<код>/LC_MESSAGES/django.po`.  
4. Выполните `python manage.py compilemessages`.

Тогда при языке `en` (и других) пользователь увидит переведённый интерфейс. Без этого переключатель и cookie по IP будут работать, но текст останется русским.

**Рекомендация:** для неизвестной страны сейчас не выставляется язык по умолчанию (остаётся `LANGUAGE_CODE = 'ru'`). Имеет смысл в middleware при неизвестной стране подставлять `en`, чтобы зарубежные пользователи по умолчанию видели английский.
